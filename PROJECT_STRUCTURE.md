# Исчерпывающая архитектура и структура проекта VKS-site

Этот документ представляет собой полную, пофайловую карту проекта. Проект построен на фреймворке **Astro 5.0** с использованием **Tailwind CSS** и следует строгим правилам: **MPA-first** (Multi-Page Application), **Zero-External** (никаких внешних запросов к CDN), и **Strict Content-Driven Architecture** (никакого хардкода текста в UI-компонентах).

---

## 🛠 Корень проекта (Конфигурация и Окружение)

Файлы в корне определяют правила работы с кодом, настройку сборки и стиль форматирования.

| Файл | Описание и роль в архитектуре |
| :--- | :--- |
| `astro.config.mjs` | **Главный конфигуратор Astro**. Настраивает интеграции (Tailwind, MDX, Sitemap), задает `base` URL для корректного деплоя на GitHub Pages (в подпапку `/vks-astro/`) или в корень (`/`) для продакшена. |
| `tailwind.config.mjs` | **Дизайн-система**. Содержит дизайн-токены (фирменные цвета `navy`, `blue`, базовые шрифты, размеры экранов, анимации). Запрещает использование плагина базовых стилей Astro (`applyBaseStyles: false`), чтобы иметь полный контроль над CSS. |
| `.cursorrules` | **Машинные директивы**. Критически важный свод правил для AI-ассистентов. Запрещает ломать архитектуру, хардкодить текст в UI, вынуждает использовать `cn()` и сортировать Tailwind-классы. |
| `ARCHITECTURE.md` | **Техническое задание**. Человекочитаемая конституция проекта. Описывает требования к производительности (Lighthouse 100), структуру слоев и запреты (например, недопустимость `<img>` вместо `<Image>`). |
| `developer_guide.md` | **Гайдлайн разработчика**. Инструкции по созданию анимаций, работе с UI-компонентами и организации Tailwind-классов (через Prettier). |
| `package.json` | **Манифест NPM**. Имена скриптов (dev, build, lint), список всех зависимостей. Здесь же настроен вызов `scripts/post-build.mjs` сразу после `astro build`. |
| `package-lock.json` | Замороженные версии зависимостей для гарантированно одинаковой сборки на любой машине. |
| `eslint.config.mjs` | Конфигурация линтера ESLint для проверки JavaScript/TypeScript. Установлен плагин `eslint-plugin-astro`. |
| `.stylelintrc.json` | Правила линтинга для CSS, чтобы соблюдать порядок свойств. |
| `.prettierrc.json` / `.prettierignore` | Настройки автоматического форматировщика. Обеспечивает строгий порядок Tailwind-классов во всех `.astro` файлах. |
| `tsconfig.json` | Настройки TypeScript. Включает базовый `astro/tsconfigs/strict` и задает красивый алиас `@/*` для папки `src/*`. |
| `.editorconfig` | Универсальные настройки редактора (2 пробела, UTF-8, LF). |
| `check_output.txt` / `astro-check*.log` | Логи автоматических проверок типизации Astro. |
| `.vscode/settings.json` | Локальные настройки редактора VS Code для проекта (привязка Prettier и линтеров). |

## ⚙️ Папка `scripts/` (Скрипты автоматизации)

*   `post-build.mjs`: Скрипт постобработки (Node.js). Запускается во время `npm run build`. Он парсит сгенерированный HTML, исправляет абсолютные пути на относительные и выявляет "потерянные" абсолютные ссылки (валидация), а также конвертирует module-скрипты в defer+IIFE для оптимизации исполнения.

---

## 🌐 Папка `public/` (Статика без обработки)

Файлы, которые веб-сервер будет отдавать "как есть", минуя сборщик Astro.
*   `VKS_PPD.pdf`: Политика обработки персональных данных.
*   `robots.txt`: Инструкции для поисковых ботов (что индексировать).
*   `images/` (папка): Может содержать старые или не-оптимизируемые картинки (архив).

## 💻 Папка `src/` (Исходный код приложения)

Полноценное приложение, разделенное на зоны ответственности.

### 🖼️ `src/assets/` — Медиа, обрабатываемые сборщиком
Изображения отсюда пропускаются через оптимизатор Astro (генерация WebP/AVIF).
*   `fonts/inter-cyrillic-400...woff2`: Набор шрифта Inter (разные начертания). Self-hosting для скорости (Zero External).
*   `fonts/manrope-cyrillic-variable.woff2`: Акцидентный вариативный шрифт Manrope.
*   `belarus-map-bg.png`: Подложка для интерактивной карты Беларуси.
*   `bg-vks.png` / `hero-bg.webp`: Фоновые изображения с градиентами.
*   `logo.webp` / `logo-white.png`: Логотипы компании для светлых/темных блоков.
*   `favicon.svg`: Иконка вкладки браузера.

### 🧩 `src/components/` — Интерфейс (UI Layer)
Строго визуальные элементы. Не содержат хардкода текстов из бизнес-логики.

#### 🏗 `layout/` (Структура сайта)
*   `CookieConsent.astro`: Компонент с логикой согласия на Cookie (GDPR/Закон РБ). Хранит выбор в `localStorage`. Настроен на работу с любым корнем сайта (base URL).
*   `Footer.astro`: Подвал сайта. Использует данные контактов из констант.
*   `Header.astro` / `LandingHeader.astro`: Разные вариации шапки (обычная и для лендингов) с десктопной навигацией.
*   `MobileMenu.astro`: Выезжающее на смартфонах полноэкранное меню навигации.

#### 🧱 `sections/` (Крупные блоки страниц)
Ожидают контент снаружи или вытягивают его из коллекций / констант.
*   `Hero.astro`: Стартовый экран (экран выше сгиба) с заголовком и кнопками действия.
*   `AboutSection.astro`: Блок "О компании" (манифест и принципы работы).
*   `LeadGen.astro`: Форма сбора лидов (подвал или конец лендинга).
*   `MapSection.astro`: Секция «Масштаб внедрений» с анимированными счетчиками цифр и интерактивной картой (или мобильным слайдером).
*   `Process.astro`: Секция "Архитектура цифровой надежности" (шаги инжиниринга).
*   `Services.astro`: Блок, который рендерит коллекцию предоставляемых услуг.
*   `Solutions.astro`: Блок презентации комплексных ИТ-решений (аккордеон/грид).
*   `UnderConstruction.astro`: Заглушка для страниц "В разработке".

#### 🧲 `ui/` (Атомарный дизайн)
Переиспользуемые кусочки. Имеют строгий `interface Props`.
*   `Button.astro`: Универсальная кнопка (primary/secondary/outline стили).
*   `FeedbackForm.astro`: Компонент самой формы с полями (используется в LeadGen).
*   `HexBadge.astro`: Декоративный бейджик (гексагон), часто для цифр.
*   `Icon.astro`: Архитектура иконок (рендерит inline-SVG по имени из `src/lib/icons.ts`). Заменяет внешние шрифты иконок.
*   `MapMarker.astro` / `MapTooltip.astro`: Логика интерактивных точек на десктопной карте (позиция, цвет, тултип).
*   `MapMobileSlider.astro`: Fallback-компонент карты для экранов мобильных телефонов (свайп карточек вместо кликов по карте).
*   `ServiceCard.astro`: Карточка, отображающая превью конкретной ИТ-услуги.
*   `SolutionAccordion.astro` / `SolutionCard.astro`: Различные варианты рендера ИТ-решений.

#### 🧪 `_sandbox/` (Песочница)
Экспериментальные концепты дизайна. В конечном билде на страницах не используются.
*   `Concept1SplitScreen.astro`, `Concept2Tabs.astro`, `Concept3Bento.astro`, `ConceptB_Underline.astro` — черновики секций решений.
*   `data.ts` — временные данные для песочницы.

### 📝 `src/content/` — База данных сайта (Data Layer)
Абсолютно все массивы текстов для услуг и решений (Astro Content Collections). Сильно типизированы.
*   `config.ts`: Определяет Zod-схемы для коллекций (какие поля обязательны в Markdown).
*   `legal/` (MDX):
    *   `cookie.mdx`: Подробный текст политики Cookie.
    *   `privacy.mdx`: Markdown-версия политики конфиденциальности.
*   `services/` (MDX): Файлы описания услуг: `consulting.mdx`, `deploy.mdx`, `equipment.mdx`, `installation.mdx`, `it-audit.mdx`, `support.mdx`.
*   `solutions/` (MDX): Файлы комплексных решений: `cloud.mdx`, `mahaon.mdx`, `network.mdx`, `office.mdx`, `power.mdx`, `security.mdx`, `servers.mdx`, `software.mdx`, `storage.mdx`, `telecom.mdx`.

### 🖼️ `src/layouts/` — Обёртки страниц
Контролируют HTML каркас и SEO.
*   `RootLayout.astro`: Формирует `<!DOCTYPE html>`, блок `<head>`, подключает `main.css`, SEO мета-теги и фавиконки. Никогда не импортируется в pages напрямую.
*   `PageLayout.astro`: Обертка почти для всех страниц. Обязательно импортирует `RootLayout`, встраивает `Header` и `Footer`, а `slot` заполняет контентом страницы. Управляет плавающей шапкой на скролле.

### 🧠 `src/lib/` — Конфигурация и Строки
Единый источник истины (Single Source of Truth) для глобальных данных. Если нужно изменить фразу в UI — иди сюда.
*   `constants.ts`: Глобальные константы: Имя сайта, SEO-дескрипшены, ссылки навигации (`NAV_LINKS`, `NAV_ANCHORS`), контактные данные (телефоны, email), переводы UI-кнопок и путь к `privacyPolicyUrl`.
*   `icons.ts`: Словарь всех SVG-путей (`d="..."`) для компонента `Icon.astro`. Никакого мусора в HTML.
*   `map-data.ts`: Координаты точек для карты внедрений, размер проектов по регионам.
*   `sections.ts`: Текстовый контент (Заголовки, описания, счетчики) для секций типа Hero, About, Map.

### 📄 `src/pages/` — Точки входа (Маршруты / Routes)
Здесь собираются компоненты в полноценные страницы.
*   `404.astro`: Страница ошибки "Страница не найдена".
*   `index.astro`: Маршрут `/` (Главная). Собирает секции Landing'а.
*   `kontakty.astro`: Маршрут `/kontakty` (Заглушка/страница).
*   `o-kompanii.astro`: Маршрут `/o-kompanii`.
*   `resheniya.astro`: Маршрут `/resheniya`.
*   `uslugi.astro`: Маршрут `/uslugi` (Архив/список услуг).
*   `uslugi/[slug].astro`: Динамический SSG-роут для отрисовки конкретной услуги по URL (генерирует статику через `getStaticPaths` из коллекции `services`).
*   `[...slug].astro`: Catch-all рутер для отрисовки MDX-документов из директорий (например, legal text).
*   `_sandbox/index.astro`: Страница песочницы (игнорируется в чистовом деплое, если настроено из-за префикса `_`).

### ⚙️ `src/scripts/` — Клиентская логика (JS/TS)
Вынесенные из компонентов куски JS, если они слишком сложные.
*   `form-init.ts`: Инициализация формы обратной связи.
*   `solutions-init.ts`: Логика показа переключателей / табов для решений.
*   `sandbox-*`: Скрипты экспериментальных секций.

### 🎨 `src/styles/` — Глобальные стили
*   `main.css`: Точка входа для Tailwind (`@tailwind base; @tailwind components; @tailwind utilities;`). Задает базовые переменные (`:root`), подключает шрифты `@font-face` (для нулевых сдвигов и мгновенного отображения), и переопределяет глобальную типографику (`@apply`).

### 🛠 `src/utils/` — Функции-помощники
*   `cn.ts`: Соединяет `clsx` и `tailwind-merge`. Решает проблему конфликта CSS-классов при передаче `class={...}` через Props компонентов. Обязательна к использованию при динамических стилях.
*   `url.ts`: Критичная утилита маршрутизации. Содержит функцию `url(path)`, которая определяет, компилируется ли сайт для корня домена или в подпапку (GitHub Actions) и возвращает корректную строку адреса. Без нее ломались бы ссылки на публичные ассеты.
